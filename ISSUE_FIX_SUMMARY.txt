ISSUE REPORTED
-----------
Preview showing 404 errors when clicking "Generate Briefing"
Cannot interact with the application
User sees error: "Backend not connected" or fails to fetch

ROOT CAUSE IDENTIFIED
--------------------
1. Frontend (Next.js) running in v0 preview
2. Backend (FastAPI) NOT running in preview
3. Frontend tries to call http://localhost:8000/api/* endpoints
4. Backend not available = 404 Not Found errors
5. No graceful fallback mechanism

SOLUTION IMPLEMENTED
-------------------
Added intelligent backend detection with automatic fallback:

✓ On page load: Check if backend is running
  - If yes: Use real API endpoints
  - If no: Use realistic mock data

✓ Mock data system:
  - 4 different personas (investor, student, founder, journalist)
  - Realistic briefing responses
  - Sample related articles with similarity scores
  - Q&A answers with citations
  - 1.5s simulated delay (realistic feel)

✓ User communication:
  - "Demo Mode" indicator when using mock data
  - "✓ Backend Connected" indicator when real API available
  - Warning banner explains what's happening
  - Clear, helpful messaging
